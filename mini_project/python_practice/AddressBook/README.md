# Python Address Book

## Project Overview (프로젝트 개요)
사용자의 이름, 전화번호, 이메일, 주소, 그룹의 정보를 입력받아 객체를 생성하고, 이를 리스트로 관리하는 프로그램입니다.
단순한 데이터 저장을 넘어 객체 지향 프로그래밍(OOP)의 핵심 개념인 클래스 분리와 인스턴스 관리를 중점적으로 구현했습니다.

## Tech Stack (사용 기술)
- Language: Python 3.11.9
- Concept: OOP (Object-Oriented Programming), List, Loop

## Class Structure (클래스 구조)
이 프로젝트는 SRP를 고려하여 3개의 클래스로 역할을 분리했습니다.
1. Addr: 연락처 데이터(이름, 번호 등)를 저장하는 데이터 객체 역할을 합니다.
2. SmartPhone: 연락처 추가, 출력, 검색, 삭제 등 실질적인 기능을 담당합니다.
3. SmartPhoneMain: 사용자 입력을 받고 프로그램을 실행하는 메인 실행(View/Controller) 역할을 합니다.

## Key Features (주요 기능)
1. 연락처 입력: 사용자로부터 정보를 입력받아 Addr 객체를 생성하고 리스트에 저장합니다.
2. 전체 출력: enumerate를 활용하여 저장된 모든 객체의 정보를 인덱스와 함께 출력합니다.
3. 검색 / 삭제 / 수정: for문과 조건문을 활용하여 리스트 내 객체를 탐색하고 상태를 변경합니다.

## What I Learned (배운 점)
이 프로젝트를 진행하며 다음과 같은 내용을 깊이 있게 학습했습니다.

1. 클래스 설계와 역할 분리
단순히 코드를 나열하는 것이 아니라, Addr(데이터), SmartPhone(기능), SmartPhoneMain(실행)으로 클래스를 나누어 구조화했습니다. 이 과정에서 "어떤 코드가 어디에 위치해야 하는가"를 고민하며 객체 지향적인 설계를 경험했습니다.

2. 인스턴스 변수와 데이터의 Life Cycle
데이터가 생성되고 소멸되는 시점(Life Cycle)과 Heap Memory에 객체가 저장되는 원리를 고민했습니다. 무조건적인 self 키워드 사용을 지양하고, 데이터의 지속성이 필요한 경우에만 인스턴스 변수를 사용하여 메모리 효율과 코드의 명확성을 높였습니다.

3. 객체 리스트 관리
리스트에 단순 문자열이 아닌 객체 자체를 저장함으로써, 데이터와 기능이 묶인 상태로 관리되는 OOP의 장점을 이해했습니다.
